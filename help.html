HelpMan to the rescue!!!

Basic Usage

  double-click (mouse) or long-press (touch-screen) on a cell to zoom into highlighted content

Searching for content

  1. Click on the "telescopic sight" icon to open the search/filter menu.
  2. Label Navigating hints:
    • Content addressing newcomers is tagged with the "101" label. If you are not in US, this number can look arbitrary. Some more info is available here.
    • "TODO" tag means incomplete content. You can skip it or try to contribute with a pull request or opening a new issue.
    • Other common labels or topics include:
      • qa: Quality assurance related topic. Things that help you do better thing in no more time.
      • low_code: Things that let you create working stuff with minimal coding or work (ussually it also mean better "QA").
      • testing: Information on how to test things work the way you expected
      • resource: external links, tools, bibliography, ...
  3. Search for matching text in any "zoom-able" element by filling the regex text input.
    The "single-line" checkbox can be chequed to search only matching text in a single-line and the "case-match" checkbox to match only if Upper/lower case match.
    NOTE: user-input is interpreted as a regular expression.
    Advanced users can profit from this feature to create powerful queries
    Normal users can ignore this feature or, if curious, learn about the magic of regular expression by clicking [here]. Just some regex simple examples: 
    Ex.1:
city.*Paris               ← Search for city then any character (.) repeated zero or more times (*) then Paris
Ex.2:
year.20\d\d               ← Search for year followed by any character (:,-,white space,...) followed by 20 and two more digits
Ex.3:
[<][^>]+@[^>]+\.[^>]+[>] ← Search email likes <alice@mail.com>

  4. Optionally click on any topic to filter out/in any non/related topics. The AND/OR checkbox let's to choose whether content must contain ALL or ANY selected topic.
    Only zoomable contant that has been previously tagged or labeled with matching topics will be searched for.
  5. Finally click on the "Search Now!" button (or Press "Enter" key) to execture the search
  6. Matching zoomable content will be highlighted while non-matching zoomable elements will be hidden by default.
  7. Browse visually around the highlighted content to see if the context matches with the search.
  8. Press the "ESC" key shortcut in your keyboard or the [show all] link to show again all cells and their relation in the context of the matched content (the "neighborhood" of matching elements)
  9. Other shortcuts:
    "z" (lower-case z): Orbit/Zoom-out
    ">" (Upper-case z): macro/Zoom-in
NOTE: When just a single zoomable element match the search its content will automatically be zoomed-in
This behaviour allows to create external links to you page that automatically opens the desired cell/content using a link similar to: [this example link]
Otherwise the different zoomable elements will be highlighted and the user will choose which one to zoom-in.
Once you click on one of the the left/right arrows icons can be clicked to move to previous/next search match, or you can close the zoom and click on another zoomable element.

Sharing content

  • The full page book can be shared as a normal web page by sharing its URL (Universal Resource Locator)
  • If you want to put the attention or highlight some particular topic inside the book,add the query parameter "topics=topic1,topic2,..". For example:
    Share the linux administration page highlighting topics related to audit_user:
    ./DevOps/linux_administration_summary.html?topics=audit_user
    Share the same page, highlight other topic (learn_x_in_y_min):
    ./DevOps/linux_administration_summary.html?topics=learn_x_in_y_min
  • The URL query parameter can be used to share a text-search. For example:
    Share the linux administration page highlighting any cell including the word "disk": ./DevOps/linux_administration_summary.html?query=disk
    Or containing the word "disk" but restricting to the topic audit_user: ./DevOps/linux_administration_summary.html?query=disk&topics=audit_user
  • By default search menu with exiting topics is diplayed. add next parameter to the URL to hide it showSearchMenu=false. For Example:
    /DevOps/linux_administration_summary.html?showSearchMenu=false
    or a search with text and topics pre-selected
    /DevOps/linux_administration_summary.html?showSearchMenu=false=false&query=ssh&topics=audit_user
  • Standard HTML id attribute can be added to link to a concrete cell. It will be automatically open. Use the standard link hash (#) like: /DevOps/linux_administration_summary.html#external_links

Editing the content
